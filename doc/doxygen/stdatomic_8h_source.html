150 // == Volatile Variable =======================================================
151 // In C, and consequently C++, the volatile keyword was intended to:
152 // - allow access to memory-mapped I/O devices
153 // - allow uses of variables between setjmp and longjmp
154 // - allow uses of sig_atomic_t variables in signal handlers.
155 //
156 // Operations on volatile variables are not atomic, nor do they establish
157 // a proper happens-before relationship for threading like with the
158 // `__asm__` inline blocks.
159 // This is specified in the relevant standards (C, C++, POSIX, WIN32), and
160 // volatile variables are not thread-safe in the vast majority of current
161 // implementations.
162 // Thus, the usage of volatile keyword as a portable synchronization mechanism
163 // is discouraged by many C/C++ groups.
